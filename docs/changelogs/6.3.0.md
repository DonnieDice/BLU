# v6.3.0 Changelog

## Updates
- **Options tabs now support a true third row** - expanded the options tab layout, reserved placeholder slots for future panels, and adjusted the container sizing so the rows do not overlap.
- **Collectibles and Loot now have reserved placeholder tabs** - added dedicated placeholder tabs for future Collectibles and Loot trigger groups alongside the new Combat placeholder tab.
- **Combat now has a placeholder tab in the options flow** - added a dedicated Combat tab in alphabetical order with a clearer coming-soon panel instead of a generic reserved slot.
- **Profiles now have a dedicated management tab** - added a new Profiles tab, then compacted it into a dropdown-based selector on the left with presets beneath it and a tighter management pane on the right.
- **Profile controls are now centralized** - create, load, rename, delete, reset, and copy-from-character actions now live in the Profiles tab and use the existing BLU profile APIs.
- **Preset support was added for quick setup flows** - `donniedice's Preset`, `Spooky Preset`, `Adventure Preset`, and `Minimal Preset` can now be applied to the active profile.
- **Debug controls were refined into the General tab** - scoped debug filtering remains backed by `modules/Debug/Debug.lua`, but the UI now lives in General as a compact grid with a header toggle and hover descriptions.
- **Delve life-credit handling is more reliable** - delve life gain/loss tracking now rechecks delayed aura stack updates so life-credit sounds trigger more consistently.
- **Sound trigger coverage and dropdown behavior were tightened** - aligned several trigger aliases across loader/registry paths and reduced oversized dropdown spacing so nested menu rows fit their text better.
- **Fixed tab button sizing on stacked rows** - removed the redundant `SetSize` call and ensured tabs in columns 2+ use the wider button width only when appropriate.
- **Improved metadata and popup safety** - added a fallback for `GetAddOnMetadata` across modern and legacy APIs, and guarded `StaticPopup` editboxes to prevent nil-index crashes.
- **Chat prefix styling is cleaner** - login and debug/error chat tags now render with white brackets around colored label text.
