# v6.5.1 Changelog

## Volume Slider Fixes
- **Fixed volume sliders not appearing for BLU game sounds**: The Lua pattern `_(med|low|high)%.ogg$` used pipe `|` which is a literal character in Lua patterns, not alternation. Replaced with three separate `:match()` calls so `hasVolumeVariants` is correctly auto-detected for sounds registered with `_med.ogg` suffixes.
- **Replaced OptionsSliderTemplate with custom track-style volume control**: Single-column panels (Housing, Quest, etc.) now use the same compact 3-position click/scroll volume control as the Combat page, ensuring visual consistency across all layouts.
- **Volume fill/thumb now uses percentage-based positioning**: Low=15%, Medium=50%, High=85% of track width. A one-shot `OnUpdate` handler re-applies visuals once the track has a real rendered width, fixing stale positioning on first display.
- **Volume label (Low/Medium/High) now appears below the slider on hover**: Hidden by default, shown only when the mouse enters the slider area. Label snugged close to the track with 2px gap.

## Layout Improvements
- **Single-column rows shrunk from 90px to 68px**: Removed the separate "Currently selected:" label line; the sound name still displays inline. Row spacing tightened from 90px to 70px steps so 4 housing options fit on the page without overlap.
- **Test buttons always align to the right side**: Both single-column and 2-column layouts now anchor the test button to the right edge of the row/container, so it stays consistent whether or not a volume slider is visible.
- **Volume slider centered between dropdown and test button**: On 2-column (Combat) pages, the volume control stretches between the dropdown right edge and test button left edge, vertically centered with the dropdown button. On single-column pages, same anchoring pattern fills the gap proportionally.
- **2-column column gap adjusted**: Column gap reduced from 10px to 8px (4px per side) for slightly tighter layout.
