## v8.0.0-alpha.1

### Framework Migration — Stage 1: Event System
- `BLU:RegisterEvent` now delegates to `RGX:RegisterEvent` (pcall combat guard)
- Fallback to original BLU event frame if RGX unavailable
- Callback signature preserved: `function(event, ...)` via wrapper

### Framework Migration — Stages 2-5
- Timer system delegates to `RGX:After` / `RGX:Every` with fallback
- Hooks and slash commands delegate to RGX with fallback
- Addon bootstrap via `RGX.Addon()` with full fallback if RGX absent
- Database fully migrated to `RGX:NewDatabase()` proxy

### Removed Duplicate Modules
- `core/systems/combat_protection.lua` (344 lines) → `RGX:QueueForCombat()`
- `core/interface/dropdown.lua` (252 lines) → `RGX:GetDropdowns()`
- `Utils:Throttle`, `Utils:Debounce`, `Utils:DeepCopy` → `RGX:Throttle/Debounce/DeepCopy`
- `Utils:SafeCall` → `RGX:QueueForCombat`
- `Sounds:MuteDefaultSounds` → `RGX:GetSound():MuteList(ids)`

### Database Proxy Stabilization (June 11)
- Fixed `__newindex` — internal proxy fields (`_guard`, `_raw`, `_defaults`, `_callbacks`, `_onSwitch`) now use `rawset` instead of writing into profile SavedVars
- Fixed `__index` — added `_guard` to `rawget` list (was missing, caused re-entrancy guard to leak to profile reads)
- Removed `BLU.db = profile` from `onProfileSwitch` — proxy must never be overwritten
- Fixed `BLU.db.currentProfile` → `BLU.db:GetActiveProfile()` in profiles.lua
- Fixed `RGX:DeepMergeDefaults(...)` → `RGX:MergeTable(...)` (function never existed)
- Fixed `Database:InitializeDatabase()` → `Database:Init()` (method did not exist)
- Fixed `ResetProfile("Default")` — removed incorrect `PROTECTED_PROFILE` block (protection kept for Delete/Rename only)
- Added `GetRawDB()` helper to profiles.lua; migrated all 16 direct `_G.BLUDB` references
- `RGX.Addon()` bootstrap now passes `opts.onSwitch` through to `NewDatabase`

### Dead Code Cleanup (June 11)
- Removed `ResetAdvancedSettings`, `RebuildDatabase` from core.lua (no callers, broken internals)
- Removed shadowed `ShowExportDialog`/`ShowImportDialog`/`ShowCharacterCopyDialog` from core.lua (database.lua versions are the proper ones)
- Removed triple `PlayTestSound` definitions — kept only `initialization.lua` version
- Removed shadowed `CreateHousingPanel` from housing.lua (sound_panel.lua version wins per load order)
- Deleted empty `housing.lua`, removed from `BLU.xml` load order
- Removed dead `GetDB`/`SetDB` shims and misleading `BLU.InitializeDatabase` from database.lua
- Removed broken `Config:ExportSettings`/`Config:ImportSettings` from config.lua (`pairs(BLU.db)` iterates proxy methods, not data; zero callers)

- `sound_panel.lua` syntax fix from dropdown migration (orphaned elseif)
