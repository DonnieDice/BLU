# v6.4.1 Changelog

## Profile Fixes
- **Default stays the fallback profile**: BLU now starts on `Default` unless the user explicitly creates or selects another profile.
- **Delete no longer regenerates character profiles**: Removing the active profile falls back to `Default` instead of auto-creating a fresh character-keyed profile.
- **Blank profile names now auto-fill**: Creating a profile with an empty name now uses the current `Character-Realm` name and adds a numeric suffix if needed.

## Release Workflow
- **Discord release summaries are cleaner**: Release notifications now send a condensed "What's New" summary for Discord while keeping the full changelog body for GitHub releases.

## Bug Fixes
- **Fixed `QUEST_TURNED_IN` playback error**: Sound playback logging now safely handles missing volume values instead of throwing `bad argument #3 to 'format'`.
