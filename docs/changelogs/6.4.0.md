# v6.4.0 Changelog

## New Features
- **Sound Output card on General tab**: Sound channel selector and channel volume slider are now a self-contained panel in the top-right of the General tab.
- **CVar-backed channel volume slider**: The volume slider reads and writes WoW's actual audio channel CVars (`Sound_MasterVolume`, `Sound_SFXVolume`, `Sound_MusicVolume`, `Sound_AmbienceVolume`, `Sound_DialogVolume`), keeping it in sync with the in-game audio options.
- **Preset confirmation dialog**: Applying a preset now shows an overwrite confirmation dialog before modifying the selected profile.
- **Tab-style animated buttons**: All Profiles panel action buttons (Create, Rename, Reset, Copy, and all preset buttons) use the same bg-texture + border + text hover animation as the tab buttons, with consistent tooltips.

## Profile Management Overhaul
- **Default profile is permanent and always available**: `Default` is pinned to the top of the profile dropdown and can be loaded, modified, and reset like any other profile — it cannot be deleted or renamed.
- **Auto character profile on first run**: BLU auto-creates a character-keyed working profile on first login instead of using Default directly.
- **Create profile starts from clean defaults**: New profiles always start from factory default settings and switch to immediately.
- **Copy produces a true independent deep copy**: Changes to a copied profile no longer affect the original.
- **Real-time UI rebuild on profile switch**: Switching profiles fully rebuilds the currently visible tab immediately; all other tabs rebuild on next view.
- **Preset apply rebuilds UI immediately**: Sound selection dropdowns reflect new preset settings without navigating away.
- **Delete falls back gracefully**: Deleting the active profile switches to any other existing profile, or auto-creates a fresh character-keyed one if none remain.
- **Default always pinned at top of dropdown**: Matches the pattern of Default/Random/None in sound dropdowns.
- **`BLUDB.activeProfile` is the authoritative profile name**: Replaced the unreliable `BLU.db.currentProfile` throughout.

## Bug Fixes
- Fixed `Slider:SetMinMaxValues(): max must be >= min` Lua error that prevented the General tab from rendering.
- Fixed "Database not ready" error shown on every attempt to open the options panel.
- Fixed all remaining `BLU.db.profile.*` schema references across the entire codebase — standardized to flat `BLU.db.*`.
- Fixed welcome message never showing due to stale nested schema check.
- Fixed debug scope filter, module enable/disable, and loader all reading from the wrong schema path.
- Fixed profile reset not resetting due to stale profile name comparison.
- Fixed preset application writing into a nested sub-table instead of the flat profile.
- Fixed ghost widget stacking — tab content is now fully cleared before each rebuild so widgets never stack on repeated profile switches.
- Fixed Rename not invalidating other tabs, leaving stale profile name labels until next navigation.
- Fixed profile dropdown showing inline delete buttons inside unrelated dropdowns (e.g., Sound Channel).
- Fixed profile copies being linked to the source — copies are now deep-copied and fully independent.
- Fixed deleted char-keyed profile regenerating a `-1` suffixed copy instead of using another available profile.
